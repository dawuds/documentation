# 00 — Foundations

The conceptual base of the ISMS. Documents in this folder establish the document model, governance, lifecycle, and the canonical regulatory mapping used by every other document in the suite.

← [Repo home](../README.md)

---

## Contents

| Document | Purpose |
|---|---|
| [document-hierarchy.md](document-hierarchy.md) | The nine-layer reference pyramid (Charter → Policy → Standard → Procedure → Guideline → Baseline → Plan → Manual → Register). What each layer is, who approves it, how stable it is. |
| [tiering-model.md](tiering-model.md) | Maturity-to-active-tiers map (M1 → M4). General Bank operates at M3 — Policy / Standard / Procedure + Plans + Registers. |
| [governance-model.md](governance-model.md) | Three-line model, approval authority per document type, review cadence, exception management, version control. |
| [document-lifecycle.md](document-lifecycle.md) | States a document moves through (Draft → In Review → Pending Approval → Approved → Effective → Under Revision → Superseded / Retired) and the gates between them. |
| [regulatory-mapping-reference.md](regulatory-mapping-reference.md) | **Binding on all other documents.** Canonical mapping for BNM RMiT (28 Nov 2025), ISO/IEC 27001 / 27002:2022, and adjacent Malaysian regulation. |

## Status

Owner: CISO (regulatory mapping co-owned with Chief Compliance Officer). Reviewed annually as part of the ISMS Management Review.

These documents are explanatory — they establish the model used by the ISMS but are not themselves mandatory **shall** statements. The mandatory statements are in [`../01-policies/`](../01-policies/).
