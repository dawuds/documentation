# documentation

A worked example of IT governance documentation structure for a Malaysian licensed bank.

## Versions

| Version | Status | Folder | Notes |
|---|---|---|---|
| **v1** | Stable | [`v1/`](v1/) | Information Security Management System (ISMS) anchored on ISO/IEC 27001:2022 + ISO/IEC 27002:2022 with BNM RMiT (28 Nov 2025) and Cyber Security Act 2024 mapping. Two worked cascades (Access Control, Incident Management). |
| **v2** | In design | (root) | Broader scope: full IT governance structure for a bank (frameworks → policies → procedures → registers). Designed top-down, simplified first. |

## Disclaimer

This documentation is illustrative. It is not legal, regulatory, or audit advice. Any organisation adopting it must validate every control against its own risk profile, regulatory obligations, and operating environment.
