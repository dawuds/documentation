## Why separated from the ISMS

The same rule that applied in v1: an ISMS reads as authoritative current state. Teaching, implementation guidance, migration plans, audience-tailored reading paths — all of these are *about* the ISMS rather than *part of* it. Keeping them in `_learning/` preserves the working-ISMS voice for the rest of the suite.
