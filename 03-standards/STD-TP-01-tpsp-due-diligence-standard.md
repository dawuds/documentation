## 1. Purpose
Specify the depth, evidence, and approval requirements for TPSP due diligence aligned with RMiT 10.47 and the Appendix 8 Risk Coverage Matrix.

## 2. Scope
All new TPSP engagements; periodic reassessment of existing material TPSPs.

## 3. Requirements

### 3.1 Risk Coverage Matrix application

Per RMiT Appendix 8, due diligence shall cover (as applicable to engagement):

| Risk dimension | Evidence sources |
|---|---|
| Financial soundness | Audited accounts last 3 years; credit rating; insurance |
| Legal and regulatory standing | Litigation history; regulatory enforcement; sanctions screening |
| Operational resilience | BCMS attestation (ISO 22301 / equivalent); BIA evidence |
| Information security | ISO 27001 certification; SOC 2 Type 2; penetration test results; security control attestation |
| Data protection | Data handling controls; data residency; cross-border safeguards |
| Sub-contracting | 4th-party identification; flow-down of obligations |
| Concentration risk | TPSP's own customer concentration; geographic concentration |
| Exit | Documented exit capability; data return / destruction; transition assistance |
| Shariah (if Islamic product-system relevant) | Shariah-compliance attestation or Shariah advisor engagement |

### 3.2 Depth by criticality

| TPSP criticality | Due diligence depth |
|---|---|
| Tier 0 (material) | Full RCM coverage; on-site review optional; CRO + CISO + CCO sign-off |
| Tier 1 (material) | Full RCM coverage; remote review acceptable; CISO sign-off |
| Tier 2 (non-material) | Documentary review per relevant RCM dimensions; Procurement sign-off |
| Tier 3 (immaterial) | Sanctions + basic verification only |

### 3.3 Evidence retention

| Ref | Requirement |
|---|---|
| 3.3.1 | All due diligence evidence retained for 7 years post-engagement end. |
| 3.3.2 | Linked to [REG-TPS](../06-registers/REG-TPS-third-party-service-provider-register.md) entry. |

### 3.4 Reassessment cadence

| TPSP criticality | Reassessment cadence |
|---|---|
| Tier 0 | Annual |
| Tier 1 | Annual |
| Tier 2 | Bi-annual |
| Tier 3 | Tri-annual |

Material adverse change (regulatory action, security incident, financial distress, M&A) triggers immediate reassessment regardless of cadence.

### 3.5 Shariah review

Where TPSP serves Islamic finance product systems, Shariah Compliance review is required at initial DD and at material change. Shariah Committee approval for Tier 0/1.

## 4. Exceptions
Per [POL-10](../02-policies/POL-10-it-vendor-management-policy.md). Reduced DD depth requires CRO approval.
