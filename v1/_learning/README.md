## Why this is separated

In a real bank ISMS, the suite reads as **authoritative current state**. It does not advertise reading orders, time estimates, audience profiles, or wish-lists of unbuilt documents — those signals are wrong in a working ISMS and can cause audit confusion ("why is the master register pointing to incomplete coverage?").

Because this repo is also a **demonstration artefact** for consulting use, the teaching overlay is preserved here, clearly separated from the ISMS proper.
