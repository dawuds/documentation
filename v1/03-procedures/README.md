## Purpose

A standard says *what specifically* is required (e.g., "identity disablement on termination shall complete within 2 hours of HR trigger"). A procedure says *how the operator delivers that*: who initiates, which system, which workflow, which evidence is produced, what SLA applies. Procedures change when tools change; policies and standards stay stable.

## Anatomy

Every SOP uses the nine-section skeleton in [`../_templates/procedure-template.md`](../_templates/procedure-template.md): Purpose · Scope · Trigger · Roles · Procedure (actor / action / tool / output / SLA per step) · Exception handling · Evidence and records · Related documents · Document control.
