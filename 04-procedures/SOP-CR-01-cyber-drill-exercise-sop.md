# SOP-CR-01 — Cyber Drill Exercise SOP

| **RMiT clause(s)** | Section 11.16 (Annual Cyber Drill Exercise — mandatory) |

## 1. Purpose
Operationalise the annual cyber drill exercise required by RMiT 11.16. Coordinates planning, scenario design, execution, debrief, and lessons.

## 2. Trigger
Annual drill cycle (calendar-driven; planned in Q1 of each year for execution in Q2–Q4). Plus ad-hoc post-incident scenario re-tests.

## 3. Procedure

### Phase 1 — Planning (12 weeks before exercise)

| # | Actor | Action | Output |
|---|---|---|---|
| 1.1 | CISO | Set exercise scope, objectives, executive participation | Drill charter |
| 1.2 | Head of SOC | Design scenario(s) — typically ransomware, supplier compromise, insider, payment-system compromise (rotated annually) | Scenario design document |
| 1.3 | CISO | Brief executive team; confirm scheduling | Calendar holds |
| 1.4 | Head of Corp Comms | Prepare communication playbook for exercise (no public disclosure) | Comms plan |
| 1.5 | External facilitator (if engaged) | Confirm scope, scenario, observation criteria | Engagement letter |

### Phase 2 — Pre-exercise (4 weeks before)

| # | Actor | Action | Output |
|---|---|---|---|
| 2.1 | Head of SOC | Final scenario lock; injects timing; observer briefing | Final exercise pack |
| 2.2 | All participants | Pre-read brief (general nature only — scenario not disclosed) | Pre-read acknowledgement |
| 2.3 | CISO | RMC notification of upcoming drill | RMC noting |

### Phase 3 — Exercise execution

| # | Actor | Action | Output |
|---|---|---|---|
| 3.1 | Exercise facilitator | Issue first inject; observe response | Exercise log |
| 3.2 | Response team | Execute per [PLN-01 IRP](../05-plans/PLN-01-incident-response-plan.md) — declare, contain, communicate, etc. | Per-IRP outputs |
| 3.3 | Observers | Record decisions, gaps, strengths against scenario design criteria | Observer notes |
| 3.4 | Facilitator | Inject additional events; escalate complexity | Per scenario |
| 3.5 | Facilitator | Declare scenario end | Exit point |

### Phase 4 — Debrief (within 5 business days)

| # | Actor | Action | Output |
|---|---|---|---|
| 4.1 | Facilitator | Hot debrief immediately post-exercise — fresh impressions | Hot debrief minutes |
| 4.2 | Observers | Structured findings against criteria | Observer report |
| 4.3 | CISO | Compile drill report — strengths, gaps, action items, owners, due dates | Drill report |
| 4.4 | RMC | Receive drill report at next meeting | RMC paper |
| 4.5 | CISO + action owners | Track action items in [REG-DRL Cyber Drill Register](../06-registers/REG-DRL-cyber-drill-register.md) to closure | Action closure |

### Phase 5 — RMiT 11.16 compliance evidence

| # | Actor | Action | Output |
|---|---|---|---|
| 5.1 | CCO | Confirm exercise satisfies RMiT 11.16 annual requirement | Compliance attestation in [REG-DRL] |
| 5.2 | Internal Audit | Confirm via the audit plan that drill was conducted per requirement | Audit evidence |

## 4. Exception handling
Exercise postponement requires CISO + COO joint approval; rescheduled within 90 days to remain compliant with annual cadence.

## 5. Evidence
Drill charter; scenario design; observer reports; drill report; action item closure evidence. Retained permanently for material drills.
